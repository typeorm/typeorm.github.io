Table decorators

  • @AbstractTable()

    Abstract table is a table that contains columns and relations for all tables that will inherit this table. Database table for the abstract table is not created.

  • @ClassTableChild(tableName?: string, options?: TableOptions)

    Special type of the table used in the class-table inherited tables.

  • @ClosureTable(tableName?: string, options?: TableOptions)

    Used on a tables that stores its children in a tree using closure deisgn pattern.

  • @EmbeddableTable()

    This decorators is used on the entities that must be embedded into the tables.

  • @SingleTableChild()

    Special type of the table used in the single-table inherited tables.

  • @Table(tableName?: string, options?: TableOptions)

    This decorator is used to mark classes that will be a tables. Database schema will be created for all classes decorated with it, and Repository can be retrieved and used for it.

  • @TableInheritance(type: "single-table"|"class-table")

    Sets what kind of table-inheritance table will use.

Column decorators

  • @Column()
    @Column(type: ColumnType)
    @Column(options: ColumnOptions)
    @Column(type: ColumnType, options: ColumnOptions)

    Column decorator is used to mark a specific class property as a table column. Only properties decorated with this decorator will be persisted to the database when entity be saved.

  • @CreateDateColumn(options?: ColumnOptions)

    This column will store a creation date of the inserted object. Creation date is generated and inserted only once, at the first time when you create an object, the value is inserted into the table, and is never touched again.

  • @DiscriminatorColumn(discriminatorOptions: { name: string, type: ColumnType })

    DiscriminatorColumn is a special type column used on entity class (not entity property) and creates a special column which will contain an entity type. This type is required for entities which use single table inheritance pattern.

  • @PrimaryColumn(options?: ColumnOptions)
    @PrimaryColumn(type?: ColumnType, options?: ColumnOptions)

    Column decorator is used to mark a specific class property as a table column. Only properties decorated with this decorator will be persisted to the database when entity be saved. Primary columns also creates a PRIMARY KEY for this column in a db.

  • @PrimaryGeneratedColumn(options?: ColumnOptions)

    Column decorator is used to mark a specific class property as a table column. Only properties decorated with this decorator will be persisted to the database when entity be saved. This column creates an integer PRIMARY COLUMN with generated set to true. This column creates is an alias for @PrimaryColumn("int", { generated: true }).

  • @UpdateDateColumn(options?: ColumnOptions)

    This column will store an update date of the updated object. This date is being updated each time you persist the object.

  • @VersionColumn(options?: ColumnOptions)

    This column will store a number - version of the entity. Every time your entity will be persisted, this number will be increased by one - so you can organize visioning and update strategies of your entity.

Relation decorators

  • @JoinColumn(options?: JoinColumnOptions)

    JoinColumn decorator used on one-to-one relations to specify owner side of relationship. It also can be used on both one-to-one and many-to-one relations to specify custom column name or custom referenced column.

  • @JoinTable(options?: JoinTableOptions)

    JoinTable decorator is used in many-to-many relationship to specify owner side of relationship. Its also used to set a custom junction table's name, column names and referenced columns.

  • @ManyToMany(typeFunction: (type?: any) => Function, options?: RelationOptions)
    @ManyToMany(typeFunction: (type?: any) => Function, inverseSide?: string|((object: T) => any), options?: RelationOptions)

    Many-to-many is a type of relationship when Entity1 can have multiple instances of Entity2, and Entity2 can have multiple instances of Entity1. To achieve it, this type of relation creates a junction table, where it storage entity1 and entity2 ids. This is owner side of the relationship.

  • @ManyToOne(typeFunction: (type?: any) => Function, options?: RelationOptions)
    @ManyToOne(typeFunction: (type?: any) => Function, inverseSide?: string|((object: T) => any), options?: RelationOptions)

    Many-to-one relation allows to create type of relation when Entity1 can have single instance of Entity2, but Entity2 can have a multiple instances of Entity1. Entity1 is an owner of the relationship, and storages Entity2 id on its own side.

  • @OneToMany(typeFunction: (type?: any) => Function, inverseSide?: string|((object: T) => any), options?: RelationOptions)

    One-to-many relation allows to create type of relation when Entity2 can have multiple instances of Entity1. Entity1 have only one Entity2. Entity1 is an owner of the relationship, and storages Entity2 id on its own side.

  • @OneToOne(typeFunction: (type?: any) => Function, options?: RelationOptions)
    @OneToOne(typeFunction: (type?: any) => Function, inverseSide?: string|((object: T) => any), options?: RelationOptions)

    One-to-one relation allows to create direct relation between two entities. Entity1 have only one Entity2. Entity1 is an owner of the relationship, and storages Entity1 id on its own side.

  • @RelationCount(relation: string|((object: T) => any))

    Holds a number of children in the closure table of the column.

  • @RelationId(relation: string|((object: T) => any))

    Special decorator used to extract relation id into separate entity property.

Subscriber and listener decorators

  • @AfterInsert()

    Calls a method on which this decorator is applied after this entity insertion.

  • @AfterLoad()

    Calls a method on which this decorator is applied after entity is loaded.

  • @AfterRemove()

    Calls a method on which this decorator is applied after this entity removal.

  • @AfterUpdate()

    Calls a method on which this decorator is applied after this entity update.

  • @BeforeInsert()

    Calls a method on which this decorator is applied before this entity insertion.

  • @BeforeRemove()

    Calls a method on which this decorator is applied before this entity removal.

  • @BeforeUpdate()

    Calls a method on which this decorator is applied before this entity update.

  • @EventSubscriber()

    Classes decorated with this decorator will listen to ORM events and their methods will be triggered when event occurs. Those classes must implement EventSubscriberInterface interface.

Other decorators

  • @DiscriminatorValue(value: any)

    If entity is a child table of some table, it should have a discriminator value. This decorator sets custom discriminator value for the entity.

  • @Embedded<T>(typeFunction: (type?: any) => ObjectType<T>)

    Property in entity can be marked as Embedded, and on persist all columns from the embedded are mapped to the single table of the entity where Embedded is used. And on hydration all columns which supposed to be in the embedded will be mapped to it from the single table.

  • @Index(options?: CompositeIndexOptions)
    @Index(name: string, options?: CompositeIndexOptions)
    @Index(name: string, fields: string[], options?: CompositeIndexOptions)
    @Index(fields: string[], options?: CompositeIndexOptions)
    @Index(fields: (object: any) => any[], options?: CompositeIndexOptions)
    @Index(name: string, fields: (object: any) => any[], options?: CompositeIndexOptions)

    Composite index must be set on entity classes and must specify entity's fields to be indexed.

  • @NamingStrategy(name?: string)

    Decorator registers a new naming strategy to be used in naming things.

  • @TreeChildren(options?: RelationOptions)

    Marks a specific property of the class as a children of the tree.

  • @TreeLevelColumn()

    Creates a "level"/"length" column to the table that holds a closure table.

  • @TreeParent(options?: RelationOptions)

    Marks a specific property of the class as a parent of the tree.
